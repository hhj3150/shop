# 친구 추천(리퍼럴) 프로그램 — 운영 원칙

> 우리(목장·운영팀)가 반드시 인지하고 있어야 할 **약속의 규칙**.
> 고객에게 한 약속이므로, 아래 규칙대로 **빠짐없이** 지킨다.
> 최종 갱신: 2026-06-06 · 구현: Phase 1(관리자 보증) 라이브.

---

## 1. 한 줄 요약

내 추천코드로 **친구가 가입해 첫 정기구독을 시작하면, 추천한 분과 친구 모두 5,000원** 혜택.

## 2. 핵심 원칙 (5가지)

1. **양쪽이 받는다** — 추천인·피추천인 각각 5,000원. 한쪽만 받는 일은 없다.
2. **진짜 결제가 있어야 성립** — 친구의 **첫 정기구독 입금이 확인된 순간** 보상이 생긴다(가입만으로는 X).
3. **약속은 기록으로 보증** — 모든 보상은 원장(장부)에 남고, 관리자 화면에 100% 노출된다. **누락 불가**.
4. **결제를 절대 방해하지 않는다** — 추천 처리에 문제가 생겨도 고객 주문·결제는 정상 진행된다.
5. **어뷰징은 막는다** — 자기추천·중복·기존 고객의 추천 수령은 차단한다.

## 3. 보상 규칙

| 항목 | 내용 |
|------|------|
| 보상액 | **추천인 5,000원 + 피추천인(친구) 5,000원** (각각) |
| 성립 시점 | 친구의 **첫 정기구독 주문이 ‘입금확인’** 될 때 |
| 지급(적용) | **현재(Phase 1)**: 관리자가 원장 보고 적용 처리 → 다음 결제/혜택으로 반영 <br> **향후(Phase 2)**: 다음 정기결제에서 **자동 차감** |
| 유효기간 | 없음(원장에 남아 사라지지 않음) |

> 금액 변경 시 **두 곳을 함께** 수정: `lib/referral.ts`의 `REFERRAL_REWARD_KRW` + DB 함수 `referral_reward_amount()`.

## 4. 성립(자격) 조건 — 보상이 생기는 경우

- 친구가 **추천코드/추천링크로 가입**했고(가입 직후 자동 등록),
- 그 친구의 **첫 정기구독 주문이 ‘입금확인’** 상태가 되면,
- → 추천 상태가 **성사(qualified)** 로 바뀌고, **양쪽에 5,000원씩 ‘지급예정’** 보상이 생긴다.

## 5. 어뷰징 방지 규칙 (보상이 안 생기는 경우)

- **자기 추천 금지** — 본인 코드로 본인이 가입할 수 없다.
- **1인 1추천인** — 한 사람은 추천인을 1명만 가질 수 있다(중복 등록 불가).
- **신규만 대상** — 이미 정기구독 이력(입금확인 이상)이 있는 회원은 추천 등록 불가.
- **결제 전엔 미성립** — 가입만 하고 결제(입금확인)가 없으면 보상은 생기지 않는다(가짜 가입 차단).

## 6. 추천코드 규칙

- 회원마다 **고유 코드 1개**(마이페이지에서 확인·생성).
- 형식: **8자리**, 헷갈리는 문자(0·O·1·I·L) 제외한 대문자+숫자.
- 추천링크: `https://shop.a2jerseymilk.com/?ref=코드`
- 링크로 들어온 방문자의 코드는 자동 저장됐다가, **가입 완료 시 자동 등록**된다.

## 7. 약속을 어떻게 보증하나 (중요)

- 획득한 모든 보상은 **불변 원장(`referral_rewards`)** 에 영구 기록된다.
- 보상 상태: **지급예정(earned) → 적용완료(applied)** / 또는 **무효(void)**.
- **관리자 화면(종합 관리 탭 → ‘친구 추천 현황’)** 에서 전부 보이고, 합계가 표시된다.
  → 빠뜨릴 수가 없는 구조. 이것이 “약속이 반드시 지켜지는” 장치다.

## 8. 관리자 운영 절차

1. **관리자 → 종합 관리 탭 → ‘친구 추천 현황’** 패널을 본다.
2. 추천 관계(추천인 → 친구), 상태(대기/성사/취소), 코드, 날짜가 보인다.
3. 보상별로 역할(추천인/친구)·금액·상태가 보인다.
4. **‘지급예정’** 보상은 [적용] 또는 [무효] 처리한다.
   - [적용]: 실제로 혜택을 줬을 때(또는 다음 결제에 반영했을 때) 눌러 ‘적용완료’로 기록.
   - [무효]: 어뷰징 등으로 취소할 때(사유 입력).
5. 상단 **‘지급예정 합계’** 로 미지급 약속 총액을 항상 확인한다.

## 9. 고객이 보는 화면

- **마이페이지 → ‘친구 추천하고 둘 다 혜택’ 카드**:
  - 내 추천코드 + [추천 링크 공유](복사/카톡)
  - 받은 보상(지급예정) 합계 · 적용 완료 합계

## 10. 운영 주의

- 보상 금액 수정은 **코드 상수 + DB 함수 두 곳 동기**(§3 참고).
- DB는 **수동 마이그레이션**(`migration-referral-program.sql`)이 적용돼 있어야 동작.
- 추천 처리는 결제와 분리돼 있어 **고객 결제를 막지 않는다**(설계상 안전).
- Phase 2(자동 차감) 도입 전까지는 관리자 [적용] 처리로 약속을 이행한다.
