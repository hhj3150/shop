# 관리자 — 회원 검색 후 개별 문자 발송 (Spec A) Design

**작성일:** 2026-06-08
**상태:** 설계 확정(사장님 "너의 권고대로" 위임 — 권고 기본값으로 확정)

## 목표

관리자가 특정 회원을 찾아 **1:1 문자**를 보낼 수 있게 한다. 주 용도: **주문 미완료/입금 안내**(가입·입금은 했는데 주문이 없는 회원에게 "구매하기를 눌러 주문을 완료하세요" 안내 — 고아 입금 문제와 직결), 일반 안내.

## 배경 — 이미 있는 것 (신규 인프라 불필요)

- **`app/api/broadcast` (Solapi `sendBulk`)**: 관리자 단체문자 API. `{message, subject, isAd, optout, recipients[]}` 수신, 서버에서 `is_admin` 재검증, 전화번호 정규화, 광고성((광고) 표기·무료수신거부·야간 21–08시 차단) 처리. **수신자 1명도 그대로 허용**(recipients 길이 ≥1).
- **`app/admin/page.tsx`**: "단체문자 발송" UI(약 1707행)가 이미 위 API를 호출 중 → 호출 패턴 재사용.
- **회원 모달**: `selectedMember` 상태 + `selectedMemberRow`/`memberSummary`(약 723–733행)로 선택 회원 상세를 이미 띄움. 회원별 `phone`은 `nameByUser`/`phoneByUser` 맵으로 보유.

→ Spec A = 위 회원 모달에 **"문자 보내기" 진입점 + 작성 패널**을 얹고, 기존 broadcast 호출을 **수신자 1명**으로 재사용하는 일.

## 설계 (확정)

**위치:** "회원·구독" 탭 → 회원 클릭 시 뜨는 **회원 모달 안**에 `문자 보내기` 버튼. 클릭 시 같은 모달 내 작성 영역(또는 경량 서브패널) 노출.

**작성 패널:**
- **수신자**: 선택 회원 이름·연락처 자동 표시(읽기전용). 회원에 `phone`이 없으면 버튼 비활성 + "연락처 없음" 안내.
- **메시지**: textarea(최대 2000자, LMS 한도 — broadcast `MAX_LEN`와 일치).
- **프리셋 2개**(누르면 textarea 채움, 자유 편집 가능):
  1. **주문 미완료 안내** — "[송영신목장] {이름}님, 가입 감사합니다. 아직 주문이 완료되지 않았어요. 주문 페이지에서 '구매하기'를 누르면 입금 계좌가 안내됩니다. 도움 필요하시면 회신해 주세요."
  2. **입금 안내** — "[송영신목장] {이름}님, 주문 입금이 아직 확인되지 않았습니다. 안내된 금액·계좌로 입금 부탁드립니다."
  - `{이름}`은 선택 회원 이름으로 **자동 치환**(프리셋 삽입 시점에 문자열 치환). 주문번호 등 그 외 치환은 v1 범위 밖(수동 편집).
- **정보성/광고성 토글**: 기본 **정보성**(isAd=false). 광고성 체크 시 broadcast 기존 규칙((광고) 표기·무료수신거부 문구·야간 차단) 그대로 적용.
- **발송**: 미리보기 확인 → `POST /api/broadcast` (세션 토큰, `recipients:[회원 phone]`, `message`, `isAd`) → 결과 토스트.

**보안/법:** 발송은 관리자 세션 토큰으로 broadcast 호출, 서버가 `is_admin` 재검증. 광고성 야간 차단·표기는 서버(기존)에서 강제. 실제 발송은 사장님이 버튼으로 1건씩.

## 단위 분리 / 테스트

- 프리셋 치환·발송 페이로드 구성을 **순수 함수**(예: `lib/admin-sms.ts`의 `fillPreset(name, memberName)`, `buildBroadcastPayload(...)`)로 분리 → Vitest 단위 테스트(React/네트워크 비의존). UI는 이 순수 함수를 호출만.

## 범위 밖 (YAGNI)

- 자동 발송·예약 발송·발송 이력 화면
- 주문번호 등 동적 토큰 자동 치환(이름만 자동, 나머지는 편집)
- 새 발송 인프라(기존 broadcast 재사용)

## 데이터 흐름

```
관리자(회원 모달) → 프리셋/입력 → 미리보기 확인
  → POST /api/broadcast {recipients:[phone], message, isAd}  (세션 토큰)
  → 서버: is_admin 재검증 → (광고성 규칙) → Solapi sendBulk(n=1)
  → 결과 토스트
```

## 파일(예상)

- **Modify** `app/admin/page.tsx` — 회원 모달에 "문자 보내기" 버튼 + 작성 패널 상태/핸들러(기존 broadcast 호출 패턴 재사용).
- **Create** `lib/admin-sms.ts` + `lib/admin-sms.test.ts` — 프리셋 치환·페이로드 순수 함수(TDD).
- broadcast API는 **수정 없음**(수신자 1명 이미 허용).

## 검증

- `vitest run`(admin-sms 포함) / `tsc --noEmit` / `next build --webpack` 통과.
- 발송 자체(Solapi)는 외부 연동이라 자동검증 불가 → 순수 함수 단위테스트 + 미리보기로 갈음, 실제 1건 발송은 사장님 확인.
